# 云服务器部署教程

## 部署流程概览

```
你的电脑（开发）
    │
    │ podman push（推送镜像）
    ▼
镜像仓库（阿里云/Docker Hub）
    │
    │ docker pull（服务器拉取）
    ▼
云服务器（部署运行）
    │
    │ 公网 IP / 域名
    ▼
任何人打开浏览器访问
```

## 二、推送镜像到仓库

### 方案 A：阿里云容器镜像服务（推荐，国内快）

1. 打开 https://cr.console.aliyun.com/
2. 创建命名空间（如 `myimagegroup`）
3. 创建两个仓库：`backend` 和 `frontend`

在本地电脑执行：
```bash
# 登录阿里云镜像仓库
podman login registry.cn-hangzhou.aliyuncs.com

# 给镜像打标签
podman tag image-group-backend registry.cn-hangzhou.aliyuncs.com/myimagegroup/backend:latest
podman tag image-group-frontend registry.cn-hangzhou.aliyuncs.com/myimagegroup/frontend:latest

# 推送镜像
podman push registry.cn-hangzhou.aliyuncs.com/myimagegroup/backend:latest
podman push registry.cn-hangzhou.aliyuncs.com/myimagegroup/frontend:latest
```

### 方案 B：Docker Hub（国外，免费）

```bash
# 登录 Docker Hub（需要注册 https://hub.docker.com/）
podman login docker.io

# 打标签（yourname 换成你的 Docker Hub 用户名）
podman tag image-group-backend docker.io/yourname/imagegroup-backend:latest
podman tag image-group-frontend docker.io/yourname/imagegroup-frontend:latest

# 推送
podman push docker.io/yourname/imagegroup-backend:latest
podman push docker.io/yourname/imagegroup-frontend:latest
```

## 三、服务器部署

### 3.1 登录服务器

```bash
ssh root@你的服务器IP
# 输入密码
```

### 3.2 安装 Docker

```bash
# Ubuntu 系统一键安装
curl -fsSL https://get.docker.com | sh
systemctl start docker
systemctl enable docker

# 安装 docker compose 插件
apt install docker-compose-plugin -y

# 验证
docker --version
docker compose version
```

### 3.3 创建项目目录

```bash
mkdir -p /root/image-group
cd /root/image-group
```

### 3.4 创建服务器版 docker-compose.yml

```bash
cat > docker-compose.yml << 'EOF'
services:
  # MongoDB 数据库
  mongo:
    image: mongo:7
    container_name: imagegroup-mongo
    ports:
      - "27017:27017"
    volumes:
      - mongo_data:/data/db
    restart: unless-stopped

  # Redis 缓存
  redis:
    image: redis:7
    container_name: imagegroup-redis
    ports:
      - "6379:6379"
    restart: unless-stopped

  # Go + Gin 后端（改成镜像地址）
  backend:
    image: registry.cn-hangzhou.aliyuncs.com/myimagegroup/backend:latest
    container_name: imagegroup-backend
    ports:
      - "8080:8080"
    environment:
      - MONGO_URI=mongodb://mongo:27017
      - REDIS_ADDR=redis:6379
    depends_on:
      - mongo
      - redis
    volumes:
      - uploads_data:/app/uploads
    restart: unless-stopped

  # Vue 前端（改成镜像地址）
  frontend:
    image: registry.cn-hangzhou.aliyuncs.com/myimagegroup/frontend:latest
    container_name: imagegroup-frontend
    ports:
      - "80:80"
    depends_on:
      - backend
    restart: unless-stopped

volumes:
  mongo_data:
  uploads_data:
EOF
```

注意：把 `registry.cn-hangzhou.aliyuncs.com/myimagegroup/` 替换成你实际的镜像仓库地址。

### 3.5 登录镜像仓库并启动

```bash
# 如果用阿里云镜像仓库
docker login registry.cn-hangzhou.aliyuncs.com

# 拉取镜像并启动
docker compose up -d

# 查看运行状态
docker ps
```

### 3.6 开放端口

在云服务器控制台的**安全组**中，添加入站规则：

| 端口 | 协议 | 说明 |
|---|---|---|
| 80 | TCP | 前端页面访问 |
| 8080 | TCP | 后端 API（可选） |

## 四、访问网站

任何人打开浏览器，输入：

```
http://你的服务器公网IP
```

就能看到文件上传网站了。

## 五、绑定域名（可选）

1. 在阿里云/腾讯云购买域名
2. 进入域名管理 → DNS 解析
3. 添加记录：
   - 类型：A
   - 主机记录：@ 或 www
   - 记录值：你的服务器 IP
4. 等待几分钟生效
5. 访问 `http://www.你的域名.com`

## 六、更新部署

以后修改了代码，更新流程：

### 在本地电脑

```bash
# 1. 修改代码

# 2. 重新构建镜像
cd D:\image-group
podman compose build

# 3. 打标签
podman tag image-group-backend registry.cn-hangzhou.aliyuncs.com/myimagegroup/backend:latest
podman tag image-group-frontend registry.cn-hangzhou.aliyuncs.com/myimagegroup/frontend:latest

# 4. 推送新镜像
podman push registry.cn-hangzhou.aliyuncs.com/myimagegroup/backend:latest
podman push registry.cn-hangzhou.aliyuncs.com/myimagegroup/frontend:latest

# 5. 同时提交代码到 Git
git add .
git commit -m "更新说明"
git push
```

### 在服务器上

```bash
cd /root/image-group

# 拉取最新镜像
docker compose pull

# 重新启动
docker compose up -d
```

## 七、常用运维命令

在服务器上执行：

```bash
# 查看所有容器状态
docker ps

# 查看日志
docker compose logs -f              # 所有服务
docker compose logs -f backend      # 只看后端

# 重启某个服务
docker compose restart backend

# 停止所有服务
docker compose down

# 停止并删除数据（慎用）
docker compose down -v

# 查看容器资源占用
docker stats
```

## 八、常见问题

### Q: 访问不了网站？
1. 检查容器是否运行：`docker ps`
2. 检查安全组是否开放了 80 端口
3. 检查日志：`docker compose logs -f`

### Q: 数据会丢失吗？
不会。MongoDB 数据存在 Docker Volume（`mongo_data`）中，重启容器不会丢失。
除非你运行 `docker compose down -v`（-v 会删除 volume）。

### Q: 如何备份数据？
```bash
# 备份 MongoDB
docker exec imagegroup-mongo mongodump --out /tmp/backup
docker cp imagegroup-mongo:/tmp/backup ./mongo-backup

# 恢复 MongoDB
docker cp ./mongo-backup imagegroup-mongo:/tmp/backup
docker exec imagegroup-mongo mongorestore /tmp/backup
```
