# 项目搭建教程

## 项目简介

Image Group 是一个文件上传系统，使用以下技术栈：
- **前端**：Vue 3
- **后端**：Go + Gin
- **数据库**：MongoDB（数据存储）+ Redis（缓存加速）
- **容器化**：Podman（兼容 Docker）
- **版本控制**：Git + GitHub

---

## 一、环境安装

### 需要安装的软件

| 序号 | 软件 | 用途 | 下载地址 |
|---|---|---|---|
| 1 | Git | 代码版本管理 | https://git-scm.com/ |
| 2 | Go 语言 | 后端开发 | https://go.dev/dl/ |
| 3 | Node.js | 前端开发（Vue 需要 npm） | https://nodejs.org/ （选 LTS） |
| 4 | Podman | 容器化（替代 Docker） | https://github.com/containers/podman/releases |
| 5 | MongoDB | 数据库 | https://www.mongodb.com/try/download/community |
| 6 | Redis | 缓存 | https://github.com/tporadowski/redis/releases |
| 7 | VS Code | 代码编辑器（推荐） | https://code.visualstudio.com/ |

### 安装验证命令
```bash
git --version       # Git
go version          # Go
node --version      # Node.js
npm --version       # npm
podman --version    # Podman
```

### MongoDB 和 Redis 安装注意
- MongoDB 安装时勾选 **Install as Service**
- Redis 安装时勾选 **Add to PATH**
- 安装后作为 Windows 服务自动运行

### Podman 初始化（首次使用）

1. WSL 2 需要更新：以管理员身份打开 PowerShell 运行 `wsl --update --web-download`
2. 重启电脑
3. 初始化 Podman 虚拟机：
```bash
podman machine init --now
```

## 三、从零搭建步骤

### 3.1 初始化 Git 仓库

```bash
cd D:\image-group
git init
```

### 3.2 创建后端项目

```bash
mkdir backend
cd backend
go mod init image-group
go get github.com/gin-gonic/gin github.com/gin-contrib/cors go.mongodb.org/mongo-driver/mongo github.com/redis/go-redis/v9
```

创建 `backend/main.go`（后端代码，包含以下 API）：

| API | 方法 | 功能 |
|---|---|---|
| /api/upload | POST | 上传文件 |
| /api/files | GET | 获取文件列表 |
| /api/files/:filename | DELETE | 删除文件 |
| /api/health | GET | 健康检查（检测 MongoDB/Redis 状态） |

后端核心逻辑：
- 接收文件 → 保存到 `uploads/` 目录
- 文件信息存入 MongoDB
- 查询时先查 Redis 缓存，没有再查 MongoDB
- 上传次数记录在 Redis 中

编译后端：
```bash
go build -o server.exe .
```

### 3.3 创建前端项目

```bash
cd D:\image-group
npm create vue@latest frontend -- --default
cd frontend
npm install
npm install axios
```

修改 `frontend/src/App.vue` 为文件上传页面，包含：
- 文件选择和上传功能
- 已上传文件列表展示
- 文件删除功能
- MongoDB/Redis 服务状态显示

### 3.4 创建 .gitignore

```
frontend/node_modules/
frontend/dist/
backend/server.exe
backend/uploads/
.DS_Store
Thumbs.db
```

### 3.5 创建 Dockerfile

**backend/Dockerfile**：
```dockerfile
FROM docker.io/library/golang:1-alpine AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN go build -o server .

FROM docker.io/library/alpine:latest
WORKDIR /app
RUN mkdir -p /app/uploads
COPY --from=builder /app/server .
EXPOSE 8080
CMD ["./server"]
```

**frontend/Dockerfile**：
```dockerfile
FROM docker.io/library/node:22-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

FROM docker.io/library/nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### 3.6 创建 docker-compose.yml

```yaml
services:
  mongo:
    image: docker.io/library/mongo:7
    ports: ["27017:27017"]
    volumes: [mongo_data:/data/db]

  redis:
    image: docker.io/library/redis:7
    ports: ["6379:6379"]

  backend:
    build: ./backend
    ports: ["8080:8080"]
    environment:
      - MONGO_URI=mongodb://mongo:27017
      - REDIS_ADDR=redis:6379
    depends_on: [mongo, redis]

  frontend:
    build: ./frontend
    ports: ["3000:80"]
    depends_on: [backend]

volumes:
  mongo_data:
```

## 四、运行方式

### 方式一：本地开发模式（推荐开发时用）

分别启动后端和前端：

```bash
# 终端 1：启动后端
cd D:\image-group\backend
go run main.go

# 终端 2：启动前端
cd D:\image-group\frontend
npm run dev
```

访问：
- 前端页面：http://localhost:5173
- 后端 API：http://localhost:8080

### 方式二：容器模式（推荐部署时用）

一键启动所有服务：

```bash
cd D:\image-group
podman compose up -d --build
```

访问：
- 前端页面：http://localhost:3000
- 后端 API：http://localhost:8080

管理命令：
```bash
podman compose down          # 停止所有服务
podman compose logs -f       # 查看日志
podman ps                    # 查看运行状态
```

## 六、后端环境变量配置

后端 main.go 支持通过环境变量配置数据库连接：

| 变量 | 默认值 | 说明 |
|---|---|---|
| MONGO_URI | mongodb://localhost:27017 | MongoDB 连接地址 |
| REDIS_ADDR | localhost:6379 | Redis 连接地址 |

本地开发时使用默认值（localhost），容器中通过 docker-compose.yml 的 environment 配置。
