# 技术知识点总结

## 一、Git - 分布式版本控制系统

### 是什么
Git 是分布式版本控制系统，用来记录代码的每一次修改历史，可以随时回溯到任意版本。
由 Linus Torvalds 于 2005 年创建。

### 核心价值
- **版本控制**：记录每次修改，可回退
- **协作**：多人同时开发，不冲突
- **备份**：每个人都有完整副本

### "分布式"含义
```
集中式（SVN）：所有人 → 连接一台中央服务器 → 服务器挂了就全完了
分布式（Git）：每个人电脑上都有完整的代码和历史记录 → 任何一台挂了都有备份
```

### 常用命令
```bash
git init                    # 初始化仓库
git clone <url>             # 克隆远程仓库
git add .                   # 添加所有修改到暂存区
git commit -m "描述"         # 提交
git push                    # 推送到远程
git pull                    # 拉取远程更新
git status                  # 查看状态
git log --oneline           # 查看提交历史
git branch                  # 查看分支
git checkout -b <分支名>     # 创建并切换分支
git merge <分支名>           # 合并分支
```

### Git 应用软件
| 类型 | 软件 |
|---|---|
| 命令行 | Git CMD、Git Bash、PowerShell |
| 图形化客户端 | GitHub Desktop、Sourcetree、TortoiseGit、GitKraken |
| 编辑器内置 | VS Code、JetBrains、Visual Studio |
| 远程托管平台 | GitHub、GitLab、Gitee |

## 二、MongoDB - 文档型数据库

### 是什么
MongoDB 是 NoSQL 文档数据库，把数据存在硬盘上，数据格式为 JSON 文档，结构灵活。

### 和 MySQL 的区别
| 对比 | MySQL | MongoDB |
|---|---|---|
| 数据格式 | 表格（行和列） | JSON 文档 |
| 结构要求 | 必须预先定义表结构 | 灵活，随时可加字段 |
| 查询语言 | SQL | MongoDB 自己的语法 |
| 适合场景 | 结构固定（财务系统） | 结构多变（商品、文章） |

### 核心概念
```
MySQL          →  MongoDB
数据库(Database) →  数据库(Database)
表(Table)       →  集合(Collection)
行(Row)         →  文档(Document)
列(Column)      →  字段(Field)
```

### 数据示例
```json
{
  "_id": "abc123",
  "name": "张三",
  "age": 25,
  "hobbies": ["篮球", "编程"],
  "address": {
    "city": "北京",
    "district": "海淀区"
  }
}
```

### 基本操作
```javascript
db.users.insertOne({ name: "张三", age: 25 })           // 插入
db.users.find({ age: { $gt: 20 } })                     // 查询
db.users.updateOne({ name: "张三" }, { $set: { age: 26 } }) // 更新
db.users.deleteOne({ name: "张三" })                     // 删除
```

### 应用场景
- 电商系统（不同商品属性差异大）
- 社交平台（用户动态、评论）
- 内容管理系统（文章、博客）
- 物联网/日志系统
- 游戏（玩家数据）

## 三、Redis - 内存数据库

### 是什么
Redis 把数据存在内存中，读写速度极快（每秒 10 万+次），适合做缓存、会话、排行榜等。

### 和 MongoDB 的区别
| 对比 | MongoDB | Redis |
|---|---|---|
| 数据存在哪 | 硬盘 | 内存 |
| 速度 | 快 | 极快（快几十倍） |
| 数据量 | 大（TB级） | 小（受内存限制） |
| 断电后 | 不丢 | 可能丢（可配置持久化） |

### 角色定位
```
MongoDB → 主仓库（存所有重要数据，安全可靠）
Redis   → 快递柜（存临时的、频繁访问的数据，速度极快）
```

### 工作流程
```
用户请求 → 查 Redis → 有数据直接返回（2ms）
                    → 没有再查 MongoDB → 结果存入 Redis → 返回（200ms）
```

### 数据类型
| 类型 | 用途 | 示例 |
|---|---|---|
| String | 缓存、计数器 | `SET name "张三"` |
| Hash | 存对象 | `HSET user:1 name "张三"` |
| List | 消息队列 | `LPUSH messages "消息1"` |
| Set | 去重集合 | `SADD tags "技术"` |
| Sorted Set | 排行榜 | `ZADD rank 100 "张三"` |

### 应用场景
| 场景 | 说明 |
|---|---|
| 缓存 | 减少数据库查询，加速响应 |
| Session | 用户登录状态管理 |
| 排行榜 | 实时排序 |
| 验证码 | 自动过期 |
| 限流/防刷 | 限制请求频率 |
| 分布式锁 | 秒杀/抢购防超卖 |

## 四、MySQL vs MongoDB vs Redis 总结

```
MySQL    → 存核心业务数据（钱、订单、用户），数据关系复杂、要求严谨
MongoDB  → 存结构灵活的数据（商品、文章、日志），数据格式多变
Redis    → 存临时/高频数据（缓存、会话、计数），要求速度极快
```

三者不是竞争关系，大多数项目会同时使用两到三个，各取所长。

## 五、Gin 框架 - Go 语言 Web 框架

### 是什么
Gin 是 Go 语言中最流行的 Web 框架，是一个代码库（不是应用也不是语言），用来快速构建 HTTP API。

### 类比
```
Go 语言  = 砖头、水泥（基础建材）
Gin 框架 = 预制的墙板、门窗（别人造好的组件）
你的项目  = 用这些组件搭建出来的房子
```

### 核心功能
- 路由管理（GET/POST/PUT/DELETE）
- 路由分组（按模块组织接口）
- 请求参数绑定和验证
- 中间件（日志、鉴权、跨域）
- JSON 响应
- 文件上传

### 代码示例
```go
r := gin.Default()
r.GET("/hello", func(c *gin.Context) {
    c.JSON(200, gin.H{"message": "Hello World"})
})
r.Run(":8080")
```

## 六、Vue - 前端框架

### 是什么
Vue 是前端框架，用来构建网页的用户界面（用户看到和操作的部分）。

### 核心特点
- **响应式**：数据变了，页面自动更新
- **组件化**：页面拆成独立组件，像搭积木
- **单文件组件**：HTML/CSS/JS 写在一个 .vue 文件里

### Gin 和 Vue 的区别
```
Vue（前端）→ 负责"长什么样"、"用户怎么操作"  → 运行在浏览器
Gin（后端）→ 负责"数据怎么处理"、"逻辑怎么跑"  → 运行在服务器
两者通过 API（HTTP 请求）通信
```

### 核心功能
- 数据绑定：`{{ message }}`、`v-model`
- 条件渲染：`v-if`、`v-show`
- 列表渲染：`v-for`
- 事件处理：`@click`、`@submit`
- 路由：Vue Router
- 状态管理：Pinia

## 七、Docker / Podman - 容器化工具

### 是什么
Docker/Podman 是容器化平台，把应用和所需环境打包在一起，确保在任何电脑上都能一致运行。

### 核心概念
```
Dockerfile（说明书）→ 构建 → Image（镜像/模板）→ 运行 → Container（容器/实例）
```

| 概念 | 比喻 |
|---|---|
| 镜像（Image） | 安装光盘 |
| 容器（Container） | 运行中的电脑 |
| Dockerfile | 安装说明书 |
| Docker Hub | 应用商店 |

### Docker vs 虚拟机
| 对比 | 虚拟机 | Docker 容器 |
|---|---|---|
| 启动速度 | 几分钟 | 几秒 |
| 占用空间 | 几 GB | 几十 MB |
| 性能损耗 | 较大 | 几乎没有 |

### Docker vs Podman
| 对比 | Docker | Podman |
|---|---|---|
| 后台守护进程 | 需要 | 不需要 |
| 管理员权限 | 需要 | 不需要 |
| 命令语法 | `docker run` | `podman run`（一样） |
| 安全性 | 一般 | 更安全 |

### 常用命令
```bash
podman ps                           # 查看运行中的容器
podman images                       # 查看本地镜像
podman run -d -p 8080:80 nginx      # 运行容器
podman stop <容器ID>                 # 停止容器
podman compose up -d                # 启动所有服务
podman compose down                 # 停止所有服务
podman compose up -d --build        # 重新构建并启动
podman compose logs -f              # 查看日志
```

## 八、各工具在项目中的角色

```
开发阶段                    运行阶段                   部署阶段
────────                   ────────                   ────────
Git ──────────── 全程记录代码变更 ──────────────────────────
Vue ──── 写前端 ──── 前端页面运行在浏览器 ──── 打包成静态文件
Gin ──── 写后端 ──── 后端服务运行在服务器 ──── 编译成可执行文件
MongoDB ──────────── 存储数据 ────────────────────────────
Redis ────────────── 缓存加速 ────────────────────────────
Podman ─────────────────────────────────── 打包部署到服务器
```

| 工具 | 角色 | 项目没有它能运行吗 |
|---|---|---|
| Git | 代码管理工具 | 能 |
| Vue | 前端框架 | 不能（没有页面） |
| Gin | 后端框架 | 不能（没有 API） |
| MongoDB | 数据存储 | 不能（没有数据库） |
| Redis | 缓存加速 | 能（但会变慢） |
| Podman | 部署工具 | 能（但部署麻烦） |
